# OOD

### 5C 解题法

**Clarify**

通过和面试官交流，去除题目中的歧义，确定答题范围

1. What
2. How
3. Who
4. When

**Core Object**

确定题目所涉及的类，以及类之间的映射关系

**Cases**

确定题目中所需要实现的场景和功能

**Classes**

通过类图的方式，具体填充题目中设计的类

**Correctness**

检查自己的设计，是否满足关键点

1. validate use cases: 检查是否支持所有的use case
2. Follow good practice
3. S.O.L.I.D
4. Design pattern
